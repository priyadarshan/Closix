Function READ

Syntax

read file-descriptor vector start end => effective-count, error

Arguments and values

file-descriptor -- A non-negative fixnum.

vector -- A vector with element-type (unsigned-byte 8).

start -- A non-negative fixnum.

end -- A non-negative fixnum.

effective-count -- A non-negative fixnum.

error -- A fixnum.

Description

The function read attempts to read up end - start bytes from the file descriptor file-descriptor, filling the corresponding indices in vector with the bytes read.

Errors

  • If file-descriptor refers to a file other than a socket and has been marked nonblocking, and the read would block, then the value of error is +eagain+.

  • If file-descriptor is not a valid file descriptor, or not open for reading, the value of error is +ebadf+.

  • If the call was interrupted by a signal before any data was read, then the value of error is +eintr+.

  • If file-descriptor is attached to an object which is unsuitable for reading; or the file was opened with the +o-direct+ flag, and either the value specified in by end - start, or the current file offset is not suitably aligned, then the value of error is +einval+.
